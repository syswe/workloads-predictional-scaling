Model Genel Bakış

PHPA, çeşitli tahmin modellerini destekleyerek farklı senaryolar için optimize edilmiş otomatik ölçeklendirme sağlar. Her model farklı güçlü yanları ve kullanım alanları vardır.

🤖 Makine Öğrenmesi Modelleri

Karmaşık desenler ve doğrusal olmayan ilişkiler için optimize edilmiş modeller.

  • GBDT: Gradient Boosted Decision Trees
  • XGBoost: Extreme Gradient Boosting
  • CatBoost: Categorical Boosting
  • VAR: Vector Autoregression

📊 İstatistiksel Modeller

Hızlı ve hafif modeller, basit desenler için idealdir.

  • Holt-Winters: Mevsimsel zaman serisi
  • Linear: Doğrusal trend tahmini
💡 Model Seçimi: Veri deseninize ve performans gereksinimlerinize göre model seçimi yapın. Belirsizlik durumunda çoklu model yaklaşımını tercih edin.

Ortak Yapılandırma

Tüm modeller aşağıdaki ortak yapılandırma seçeneklerini paylaşır:

Parametre Tür Açıklama Varsayılan
type string Model türü (GBDT, XGBoost, vb.) -
name string Benzersiz model adı -
perSyncPeriod int Kaç sync periyodunda bir çalışacağı 1
calculationTimeout int (ms) Hesaplama zaman aşımı 30000
startInterval duration Model başlangıç zamanlaması -
resetDuration duration Veri sıfırlama süresi -

Örnek Temel Konfigürasyon

models:
  - type: GBDT
    name: my-gbdt-model
    perSyncPeriod: 1          # Her sync periyodunda çalış
    calculationTimeout: 25000  # 25 saniye timeout
    startInterval: 1m         # Dakika başlarında başla
    resetDuration: 24h        # 24 saat sonra veriyi sıfırla

Makine Öğrenmesi Modelleri

GBDT (Gradient Boosted Decision Trees)

Scikit-learn tabanlı gradient boosting modeli. Kısa-orta vadeli tahminlerde yüksek doğruluk sağlar ve hızlı eğitim süresine sahiptir.

⚙️ Konfigürasyon Parametreleri

Parametre Tür Açıklama Varsayılan
lookAhead int (ms) Tahmin ufku (milisaniye) 10000
historySize int Saklanacak geçmiş veri sayısı 30
lags int Özellik olarak kullanılacak geçmiş değer sayısı 6

📝 Örnek Kullanım

models:
  - type: GBDT
    name: gbdt-predictor
    gbdt:
      lookAhead: 10000    # 10 saniye önceden tahmin
      historySize: 30     # 30 veri noktası sakla
      lags: 6             # Son 6 değeri özellik olarak kullan
✅ En İyi Kullanım Alanları:
  • Kısa vadeli yük tahminleri (10-60 saniye)
  • Düzenli günlük desenlere sahip uygulamalar
  • CPU kullanımında ani değişiklikler gösteren sistemler

🔧 Dahili Model Parametreleri

GBDT modeli şu scikit-learn parametreleriyle çalışır:

GradientBoostingRegressor(
    n_estimators=150,      # Ağaç sayısı
    learning_rate=0.1,     # Öğrenme oranı
    max_depth=3,           # Maksimum ağaç derinliği
    subsample=1.0,         # Alt örnekleme oranı
    random_state=42        # Sabit seed
)

XGBoost (Extreme Gradient Boosting)

XGBoost kütüphanesi kullanan gelişmiş gradient boosting modeli. Karmaşık desenler ve büyük veri setleri için optimize edilmiştir.

⚙️ Konfigürasyon Parametreleri

Parametre Tür Açıklama Varsayılan
lookAhead int (ms) Tahmin ufku 10000
historySize int Geçmiş veri boyutu 30
lags int Lag özellik sayısı 6

📝 Örnek Kullanım

models:
  - type: XGBoost
    name: xgboost-predictor
    xgboost:
      lookAhead: 15000    # 15 saniye önceden
      historySize: 50     # Daha fazla geçmiş veri
      lags: 8             # 8 lag özelliği

🔧 Dahili Model Parametreleri

XGBoost Parametreleri:
objective: 'reg:squarederror'  # Regresyon objektifi
eval_metric: 'rmse'            # RMSE metriği
max_depth: 3                   # Ağaç derinliği
eta: 0.2                       # Öğrenme oranı
subsample: 1.0                 # Alt örnekleme
colsample_bytree: 1.0          # Özellik örnekleme
tree_method: 'hist'            # Hızlı histogram yöntemi
num_round: 80                  # Boosting round sayısı
⚠️ Dikkat: XGBoost daha fazla bellek kullanır ve eğitim süresi GBDT'den uzundur. Yüksek performans gerektiren sistemlerde tercih edin.

CatBoost (Categorical Boosting)

Yandex tarafından geliştirilen CatBoost modeli. Kategorik veriler ve küçük veri setleri için özellikle etkilidir.

⚙️ Konfigürasyon Parametreleri

Parametre Tür Açıklama Varsayılan
lookAhead int (ms) Tahmin ufku 10000
historySize int Geçmiş veri boyutu 30
lags int Lag özellik sayısı 6

📝 Örnek Kullanım

models:
  - type: CatBoost
    name: catboost-predictor
    catboost:
      lookAhead: 10000
      historySize: 25     # CatBoost az veriyle de iyi çalışır
      lags: 6

🔧 Dahili Model Parametreleri

CatBoostRegressor(
    iterations=200,        # İterasyon sayısı
    learning_rate=0.1,     # Öğrenme oranı
    depth=4,               # Ağaç derinliği
    loss_function='RMSE',  # Loss fonksiyonu
    verbose=False,         # Sessiz mod
    random_seed=42         # Sabit seed
)
✅ CatBoost Avantajları:
  • Hızlı eğitim ve düşük overfitting
  • Az veri ile bile kararlı sonuçlar
  • Otomatik hiperparametre optimizasyonu
  • Kategorik özelliklerin otomatik işlenmesi

VAR (Vector Autoregression)

Statsmodels tabanlı çok değişkenli zaman serisi modeli. Zaman özelliklerini (saat, gün) otomatik olarak türeterek tahmin yapar.

⚙️ Konfigürasyon Parametreleri

Parametre Tür Açıklama Varsayılan
lookAhead int (ms) Tahmin ufku 10000
historySize int Geçmiş veri boyutu 30
lags int VAR lag order (maksimum) 6

📝 Örnek Kullanım

models:
  - type: VAR
    name: var-predictor
    var:
      lookAhead: 12000
      historySize: 40     # VAR daha fazla geçmiş veri tercih eder
      lags: 8             # Maksimum lag sayısı

🔧 Özellik Türetme

VAR modeli aşağıdaki özellikleri otomatik olarak oluşturur:

DataFrame Yapısı:
- replicas: Pod replica sayısı (hedef değişken)
- hour: Saat bilgisi (0-23)
- day_of_week: Haftanın günü (0-6)
💡 VAR Kullanım İpuçları:
  • Mevsimsel desenler için idealdir
  • Haftalık/günlük periyodik veriler için optimize
  • Çok değişkenli bağımlılıkları yakalar
  • Minimum 20-30 veri noktası gerektirir

İstatistiksel Modeller

Holt-Winters Exponential Smoothing

Mevsimsel zaman serileri için klasik istatistiksel model. Alpha, beta, gamma parametreleriyle trend ve mevsimselliği modeller.

⚙️ Temel Parametreler

Parametre Tür Açıklama Aralık
alpha float Seviye düzgünleştirme katsayısı 0.0-1.0
beta float Trend düzgünleştirme katsayısı 0.0-1.0
gamma float Mevsimsellik düzgünleştirme katsayısı 0.0-1.0
seasonalPeriods int Bir mevsimin sync period sayısı >0
storedSeasons int Saklanacak mevsim sayısı >0
trend string Trend tipi (additive/multiplicative) -
seasonal string Mevsimsellik tipi (additive/multiplicative) -

📝 Örnek Kullanım

models:
  - type: HoltWinters
    name: holt-winters-predictor
    startInterval: 60s        # Dakika başlarında başla
    holtWinters:
      alpha: 0.9              # Yüksek seviye duyarlılığı
      beta: 0.1               # Düşük trend duyarlılığı
      gamma: 0.3              # Orta mevsimsellik
      seasonalPeriods: 6      # 6 sync period = 1 mevsim
      storedSeasons: 4        # 4 mevsim saklasını
      trend: additive         # Toplama trend
      seasonal: additive      # Toplama mevsimsellik

🔄 Dinamik Parametre Ayarlama

Holt-Winters parametrelerini runtime'da HTTP endpoint'ten alabilirsiniz:

holtWinters:
  runtimeTuningFetchHook:
    type: "http"
    timeout: 2500
    http:
      method: "GET"
      url: "http://tuning-service/holt_winters"
      successCodes:
        - 200
      parameterMode: body   # Büyük veriler için body kullan
  # Fallback değerleri
  alpha: 0.9
  beta: 0.1
  gamma: 0.3
  seasonalPeriods: 6
  storedSeasons: 4
  trend: additive
  seasonal: additive
✅ Holt-Winters İdeal Durumlar:
  • Saatlik/günlük düzenli desenlere sahip uygulamalar
  • E-ticaret siteleri (öğle/akşam yoğunlukları)
  • İş uygulamaları (çalışma saatleri deseni)
  • Hızlı hesaplama gereken durumlar

Linear Regression

Basit doğrusal trend tahmini için hızlı ve hafif model. Aniden değişen trendleri yakalamak için idealdir.

⚙️ Konfigürasyon Parametreleri

Parametre Tür Açıklama Varsayılan
lookAhead int (ms) Tahmin ufku 10000
historySize int Kullanılacak geçmiş veri sayısı 6

📝 Örnek Kullanım

models:
  - type: Linear
    name: linear-predictor
    linear:
      lookAhead: 5000       # 5 saniye önceden
      historySize: 10       # Son 10 veriyi kullan
⚠️ Linear Model Sınırlamaları:
  • Sadece doğrusal trendleri yakalar
  • Mevsimsellik modelleyemez
  • Ani değişikliklerde yavaş adapte olur
  • Basit senaryolar için uygundur

Çoklu Model Stratejileri

Birden fazla modeli kombine ederek daha güvenilir tahminler elde edebilirsiniz. PHPA dört farklı karar stratejisi sunar:

📈 Maximum

En yüksek tahmini seçer. Konservatif ölçeklendirme için idealdir.

decisionType: maximum

📉 Minimum

En düşük tahmini seçer. Kaynak tasarrufu için kullanılabilir.

decisionType: minimum

📊 Mean

Tüm tahminlerin ortalamasını alır. Dengeli yaklaşım sağlar.

decisionType: mean

📊 Median

Medyan değeri kullanır. Aykırı değerlere karşı daha dayanıklıdır.

decisionType: median

📝 Çoklu Model Örneği

apiVersion: syswe.me/v1alpha1
kind: PredictiveHorizontalPodAutoscaler
metadata:
  name: multi-model-phpa
spec:
  scaleTargetRef:
    apiVersion: apps/v1
    kind: Deployment
    name: my-app
  minReplicas: 1
  maxReplicas: 20
  decisionType: mean  # Modellerin ortalamasını al
  models:
    - type: GBDT
      name: gbdt-short-term
      gbdt:
        lookAhead: 10000
        historySize: 30
        lags: 6

    - type: XGBoost
      name: xgboost-long-term
      xgboost:
        lookAhead: 20000
        historySize: 50
        lags: 8

    - type: CatBoost
      name: catboost-stable
      catboost:
        lookAhead: 15000
        historySize: 40
        lags: 7

    - type: HoltWinters
      name: seasonal-patterns
      holtWinters:
        alpha: 0.8
        beta: 0.2
        gamma: 0.4
        seasonalPeriods: 12
        storedSeasons: 3
        trend: additive
        seasonal: additive
✅ Çoklu Model Avantajları:
  • Tek model hatalarına karşı koruma
  • Farklı zaman ufuklarında optimizasyon
  • Daha kararlı ve güvenilir tahminler
  • Çeşitli veri desenlerini yakalama

Performans Karşılaştırması

Farklı modellerin tipik performans karakteristikleri:

Model Eğitim Hızı Tahmin Hızı Bellek Kullanımı Doğruluk En İyi Senaryo
GBDT 🟡 Orta 🟢 Hızlı 🟡 Orta 🟢 Yüksek Kısa vadeli tahmin
XGBoost 🟡 Orta-Uzun 🟡 Orta 🔴 Yüksek 🟢 Çok Yüksek Karmaşık desenler
CatBoost 🟢 Hızlı 🟢 Hızlı 🟡 Orta 🟢 Yüksek Az veri, kararlı sonuç
VAR 🟢 Hızlı 🟢 Hızlı 🟢 Düşük 🟡 Orta Mevsimsel veriler
Holt-Winters 🟢 Çok Hızlı 🟢 Çok Hızlı 🟢 Çok Düşük 🟡 Orta-Yüksek Düzenli mevsimsellik
Linear 🟢 Çok Hızlı 🟢 Çok Hızlı 🟢 Çok Düşük 🟡 Orta Basit doğrusal trendler

🎯 Model Seçim Rehberi

🚀 Hızlı Başlangıç

Önerilen: CatBoost veya GBDT

  • Kolay kurulum
  • İyi varsayılan parametreler
  • Çoğu senaryoda etkili

Önerilen: Çoklu Model (GBDT + XGBoost + CatBoost)

  • En yüksek doğruluk
  • Hata toleransı
  • Çeşitli desenleri yakalar

⚡ Kaynak Tasarrufu

Önerilen: Holt-Winters veya Linear

  • Düşük bellek kullanımı
  • Hızlı hesaplama
  • Basit desenler için yeterli
